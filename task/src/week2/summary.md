## 정리
> 3가지 모델로 각각 성능을 평가해보는 이유는 같은 분류 문제라도 모델이 데이터를 보는 방식이 다르기 때문 즉 “모델마다 결과와 성향이 다르다”는 걸 직접 확인하려고


### 각 모델별 차이
1. Logistic Regression
- 데이터를 선형적으로 나눠서 분류하려는 모델 입력값들의 가중치를 조합해서 확률처럼 계산한 뒤 분류 구조가 단순하고 해석이 쉬움 대신 경계가 복잡하면 한계가 있음
- 느낌: “직선/평면으로 나눌 수 있으면 잘함”

---

2. Decision Tree
- 질문을 계속 나누는 방식
- 예: radius가 얼마 이상인가?, texture가 얼마 이하인가?
- 사람이 이해하기 쉬운 규칙 형태 | 데이터에 따라 과하게 외우는(overfitting) 경향이 있음
= 느낌: “조건문으로 분기하면서 분류”

---

3. Random Forest
- 여러 개의 Decision Tree를 만들어서 다수결로 결정 / 트리 하나보다 보통 더 안정적이며 과적합을 줄이는 데 도움됨
- 대신 개별 규칙 해석은 트리 하나보다 어려움
- 느낌: “나무 한 그루가 아니라 숲 전체 의견으로 결정”

---

> 같은 데이터라도 성능이 다를 수 있음
어떤 모델이 더 잘 맞는지 직접 비교 가능
머신러닝은 “하나의 정답 모델”이 있는 게 아니라 비교가 중요함
코드 뼈대는 비슷하고 모델만 바꾸면 되니까, 비교 학습에 좋음
교수님 수업 흐름으로 보면 특히 중요한 건 이겁니다.

> 데이터 준비 과정은 거의 비슷함 fit -> predict -> evaluate도 거의 같음
결국 바뀌는 건 모델 그래서 모델만 바꿨을 때 결과가 어떻게 달라지는지 보는 게 핵심

- Logistic Regression = 단순하고 기본적인 선형 분류
- Decision Tree = 규칙 기반 분기 분류
- Random Forest = 여러 트리의 다수결 분류
